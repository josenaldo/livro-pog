# OPÇÕES PARA CAPAS INTERNAS DE PARTE
# =============================================================================
#
# Tenho 3 opções para implementar as capas de parte. Escolha uma.
#
# ─────────────────────────────────────────────────────────────────────────────
# OPÇÃO A — \part{} nativo do LaTeX  [RECOMENDADA para PDF]
# ─────────────────────────────────────────────────────────────────────────────
#   O que é:
#     O LaTeX `book` gera automaticamente uma página de parte com número romano
#     (Parte I, Parte II...) e o título. É personalizável via `titlesec`.
#
#   O que precisa informar:
#     Nada além do que já está aqui (a linha "PARTE: Nome").
#     O gerador injeta `\part{Nome}` no markdown combinado.
#
#   Aparência padrão:
#     Página em branco + página com "Parte I" + título centralizado.
#     Pode customizar cor de fundo, imagem, etc. via \titleformat{\part}.
#
#   Limitação:
#     No EPUB, `\part` vira apenas um heading grande — sem "capa interna" visual.
#
# ─────────────────────────────────────────────────────────────────────────────
# OPÇÃO B — Arquivo markdown dedicado por parte  [MELHOR para PDF + EPUB]
# ─────────────────────────────────────────────────────────────────────────────
#   O que é:
#     Cada parte tem um arquivo .md próprio (ex: content/capitulos/parte-introducao.md)
#     com frontmatter `type: part`. O gerador cria uma página visual completa:
#     imagem de fundo, título, descrição da parte — tanto no PDF quanto no EPUB.
#
#   O que precisa informar (por arquivo de parte):
#     ---
#     title: Introdução
#     type: part                    ← obrigatório
#     part_number: I                ← ex: I, II, III
#     description: "Texto breve sobre esta parte do livro"
#     cover_image: /images/partes/introducao.png   ← opcional, imagem da parte
#     status: done
#     order_number: 1900
#     ---
#
#   Vantagem: cada parte pode ter imagem própria, texto de abertura e funciona
#             bem em PDF, EPUB e na leitura no site.
#
# ─────────────────────────────────────────────────────────────────────────────
# OPÇÃO C — Campo `part` no frontmatter do primeiro capítulo de cada parte
# ─────────────────────────────────────────────────────────────────────────────
#   O que é:
#     Sem arquivos extras. Quando o gerador encontra um capítulo com
#     `part: "Nome da Parte"` no frontmatter, injeta a capa de parte antes dele.
#
#   O que precisa informar (no frontmatter do primeiro CAP de cada parte):
#     part: "Introdução"            ← nome da parte
#     part_number: I                ← opcional
#
#   Vantagem: mais simples, sem criar novos arquivos.
#   Limitação: a capa de parte não tem descrição nem imagem própria.
#
# =============================================================================
# QUAL ESCOLHER?
#
#   → Só PDF importa, quer algo rápido?         → OPÇÃO A
#   → Quer capas ricas com imagem/texto?        → OPÇÃO B
#   → Não quer criar novos arquivos?            → OPÇÃO C
#
# Depois que você editar a estrutura acima e escolher a opção de PARTE,
# eu implemento o gerador para ler este arquivo e montar o ebook.
